To setup Pants in your repo: 

      curl -L -O https://pantsbuild.github.io/setup/pants
      chmod +x pants
The only requirement is that you have a python 2.7 interpreter somewhere on your PATH.

The first time you run the new ./pants script, it will install the latest Pants version and generate a config file pants.ini for you. On subsequent runs, the script will find the already installed Pants and run it.

If you would like to use Pants plugins published to PyPi, you can configure them as follows in pants.ini: 

      [GLOBAL]
      pants_version: 1.14.0

      plugins: [
          'pantsbuild.pants.contrib.go==%(pants_version)s',
          'pantsbuild.pants.contrib.scrooge==%(pants_version)s',
        ]
Pants will notice you changed your plugins and install them.

NB: The formatting of the plugins list is important; all lines below the `plugins:` line must be indented by at least one white space to form logical continuation lines. This is standard for Python ini files, see RFC #822 section 3.1.1 for the full rules Python uses to parse ini file entries.
